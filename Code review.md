# Code Review

## Make a clear process

Through sequence and tools

## Focus on

## Functionality

Does the code behave as the PR author likely intended? Does the code behave as
users would expect? Software design: Is the code well designed and fitted to the
surrounding architecture?

## Complexity

Would another developer be able to easily understand and use the code? Tests:
Does the PR have correct and well designed automated tests?

## Naming

Are names for variables, functions, etc. Descriptives?

## Comments

Are the comments clear and useful?

## Documentation

Did the author also update relevant documentation?
